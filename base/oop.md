# 面向对象

在生活中，当我们遇到某个要解决的问题，如果我们能很好地对问题进行抽象，那要解决这个问题就会变得很简单。

汇编语言抽象于底层机器，而命令式语言对汇编语言的抽象。但是这两种语言在使用时的考虑，都是基于计算机的结构，而不是基于所解决的问题。程序员必须先想好这个业务怎么解决，然后这个解决方案在机器上怎么运行。

而Lisp和APL将``这个解决方案在机器上怎么运行``转化成``针对待解决的问题建模``，Lisp说所有的问题都是列表，而APL说所有问题都是算法。PROLOG说所有问题转化为决策链。

面向对象的表现元素方式，对``业务怎么解决``做了进一步的提升。它是通过问题的角度来描述问题。  

- 万物皆对象

  任何问题都可以概念化为程序中的对象。

- 程序是对象的集合

  对象通过彼此发送消息来告知对方彼此所要做的事情。这里的消息就是对方法的调用请求。

- 每个对象都有其他对象所构成的存储

  B对象是可以作为A对象的属性，包含在A对象中的，这样就将B对象的复杂性隐藏起来

- 每个对象有一个type（类别）

  对象是class的实例

- 统一类型可以接收相同的信息

  从同一个类中实例化的不同对象具有相同的行为
